Core Ideas

Straightforward data modeling

Previous 52Â°North Sensor web components used a shared datamodel that aimed to allow for interoperability of the existing components. It showed that while this is a very desirable feature, it incurs disproportionate amounts of complications as the data models defined by the standards differ heavily.

To avoid such complications, the Connected Systems Implementation will not be interoperable with the other 52N SW-components. Data is stored as close to the standard-mandated form as possible, allowing for very straighforward mapping.

Technology

Webserver

Quart is used for handling all HTTP-based traffic.

Data Storage

For data storage two different storage providers are used: Elasticsearch for storage of metadata, Timescaledb for storage of observation data.

Elasticsearch

  • Well suited for loosely structured metadata

    • very flexible data models, no strict schema

    • native searchability

    • native scalability (delegated to storage provider Apache Lucene)

Timescaledb

  • Well suited for structured data

    • high performance when handling many rows of structured data

    • many inbuilt operations (spatial filtering, time-based filtering) -
